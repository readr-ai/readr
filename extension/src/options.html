DeDi directory hosting

Publish your issuer public key to a decentralized directory

By default your credentials carry an anonymous did:jwk. Publishing your public key to a DeDi namespace binds that key to a named organisation, so verifiers can confirm who issued a document — not just that the signature is internally consistent. See the DeDi docs and OpenCred docs.

Create one in your DeDi account (/dedi/api-keys). Sent as Authorization: Bearer …, stored locally, never published.

Advanced — endpoint templates (DeDi v2 defaults)

Placeholders: {namespace}, {registryName}, {recordName}. Publishing posts to save-record with ?publish=true.

Publishing auto-creates the key registry if needed. Use Create namespace first only if you don't already have one.
